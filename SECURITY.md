# Security Policy

## Reporting a vulnerability

If you discover a security issue in this software, please report it **privately**. Do **not**
open a public GitHub issue for security vulnerabilities.

- **Email:** support@jgsystemsconsulting.com
- Include: a description, reproduction steps, and the affected version (see `RELEASE-INFO.txt`).

## What to expect

- Acknowledgement within **5 business days**.
- An assessment and, where applicable, a remediation plan.
- Coordinated disclosure once a fix is available.

This is open-source software released under the MIT License. There is no paid bug-bounty
program, but we appreciate responsible disclosure and will credit reporters who request it.
